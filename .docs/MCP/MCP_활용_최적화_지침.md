# MCP 서버 활용 최적화 지침 (단독 모드)

## 🎯 목적
이 문서는 현재 단독으로 활성화된 **Context7 MCP 서버**를 집중적으로 활용하여 개발 효율을 높이는 방법을 제시합니다.

> **⚠️ 중요 변경**: 과부하 문제로 인해 Context7 외의 모든 MCP 서버(MySQL, Shell, Filesystem 등)는 비활성화되었습니다.

---

## 📋 Context7 집중 활용 전략

### 1️⃣ 코드베이스 이해 및 네비게이션

**Context7**은 전체 코드베이스의 구조와 의미를 이해하고 있습니다. 파일 내용을 직접 읽는 것보다 더 빠르고 정확하게 코드의 위치와 기능을 찾아냅니다.

#### 활용 시나리오
- **기능 검색**: "비밀번호 해싱을 처리하는 로직이 어디에 있어?"
- **영향도 분석**: "이 코드를 수정하면 영향을 받는 다른 함수들을 알려줘"
- **구조 파악**: "backend 디렉토리의 파일 구조와 각 파일의 역할을 요약해줘"

#### 추천 명령어
```
✅ "signal_history 테이블을 사용하는 모든 파이썬 함수를 찾아줘"
✅ "KIS API 토큰 갱신 로직이 구현된 위치와 방식을 설명해줘"
✅ "프론트엔드에서 API 호출을 담당하는 컴포넌트들을 나열해줘"
```

---

### 2️⃣ 문서화 및 설명 생성

코드 분석 결과를 바탕으로 문서를 자동으로 생성하거나 복잡한 로직을 자연어로 설명받을 수 있습니다.

#### 활용 시나리오
- **함수 주석 작성**: "run_analysis 함수에 대한 Docstring을 작성해줘"
- **기술 문서 작성**: "현재 시스템의 데이터 흐름도를 설명하는 마크다운 문서를 만들어줘"

#### 추천 명령어
```
✅ "db.py 파일의 주요 함수들에 대한 설명 문서를 작성해줘"
✅ "현재 SOXL 매매 전략 로직을 단계별로 풀어서 설명해줘"
```

---

## 🚫 제한 사항 및 대안

다른 MCP 서버가 비활성화되었으므로, 다음과 같은 작업은 Antigravity의 기본 도구(Native Tools)를 직접 사용해야 합니다.

| 작업 유형 | 기존 (비활성화) | **현재 대안 (Native Tools)** |
|---|---|---|
| **파일 읽기/수정** | Filesystem MCP | `view_file`, `replace_file_content` 도구 사용 |
| **명령어 실행** | Shell MCP | `run_command` 도구 사용 |
| **DB 조회** | MySQL MCP | (직접적 대안 없음, 필요 시 `run_command`로 mysql 클라이언트 사용) |
| **웹 검색** | Brave Search MCP | `search_web` 도구 사용 |
