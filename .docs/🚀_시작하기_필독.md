## 📌 핵심 원칙 (반드시 준수)

### 1️⃣ 한국어 사용 원칙
```
✅ 모든 설명, 주석, 문서는 한국어로 작성
✅ 변수명, 함수명은 영어 사용 (코드 표준)
✅ 커밋 메시지는 한국어로 작성
```

### 2️⃣ MCP 서버 활용 원칙
```
✅ 모든 작업에 MCP 서버 자동 활용
✅ 코드 검색: Context7 MCP
✅ 데이터 조회: MySQL MCP
✅ 파일 작업: Filesystem MCP
✅ 서버 관리: Shell MCP
✅ 복잡한 분석: Sequential Thinking MCP
```

### 3️⃣ 작업 완료 후 필수 절차 (⭐ 매우 중요)
```
작업 완료 → Git 버전 업데이트 → 커밋 & 푸시 → 웹 배포 → 버전 업데이트 → 요청사항 기록
```

**상세 절차**:
1. ✅ **Git 버전 업데이트**
   ```bash
   # CHANGELOG.md 업데이트
   # 버전 번호 증가 (예: v3.2.1 → v3.2.2)
   ```

2. ✅ **Git 커밋 & 푸시**
   ```bash
   git add .
   git commit -m "[v3.2.2] 작업 내용 설명"
   git push origin main
   ```

3. ✅ **웹 배포**
   ```bash
   cd /home/blue/blue/my_project/money
   ./deploy.sh
   # 또는
   web-run100
   ```

4. ✅ **웹 카피라이트 버전 업데이트**
   - 파일: `frontend/index.html` (또는 해당 레이아웃 파일)
   - 위치: Footer 영역
   - 형식: `© 2026 Stock Trading System v3.2.2`
   - Git 버전과 동일하게 업데이트

5. ✅ **요청사항 페이지에 기록**
   - 웹 페이지: `http://localhost:5000/requests` (또는 배포 URL)
   - 데이터베이스: `user_requests` 테이블
   - 기록 내용: 요청 내용, 구현 사항, 완료 시간
